# Claude Code Build Instructions: TalentAgent Agentic Layer

## Context

The base TalentAgent app is already built and running. It has:
- Supabase auth with candidate and recruiter roles
- Candidate rich profile builder (7 sections)
- Recruiter buyer agent creation form
- Anthropic-powered fit assessment API route
- Fit report display

This document adds the agentic layer on top of that foundation. Do not modify what already works unless explicitly instructed.

---

## What You Are Adding

1. **GitHub OAuth integration** -- candidates connect GitHub, system ingests their entire coding history and builds a structured technical fingerprint using Claude
2. **Technical fingerprint** -- a rich, structured representation of a developer's actual skills derived from real code, not self-reporting
3. **Job market crawler** -- autonomous agent that monitors job postings from major boards and company career pages
4. **Autonomous matching engine** -- continuously runs candidate fingerprints against crawled job postings without human intervention
5. **Candidate-facing opportunity feed** -- candidates see ranked opportunities their agent surfaced, with evidence
6. **Autonomous outreach layer** -- agent drafts and sends introduction emails to companies on behalf of matched candidates
7. **Real-time notification system** -- candidates get notified when their agent finds strong matches

## New API Routes to Build

### 1. GitHub OAuth Flow

**`app/api/github/connect/route.ts`**
Redirects user to GitHub OAuth authorization page.

```typescript
import { NextRequest, NextResponse } from 'next/server'

export async function GET(request: NextRequest) {
  const clientId = process.env.GITHUB_CLIENT_ID
  const redirectUri = process.env.GITHUB_REDIRECT_URI
  const scope = 'read:user,repo'
  
  const githubAuthUrl = `https://github.com/login/oauth/authorize?client_id=${clientId}&redirect_uri=${redirectUri}&scope=${scope}`
  
  return NextResponse.redirect(githubAuthUrl)
}
```

**`app/api/github/callback/route.ts`**
Handles the OAuth callback, exchanges code for token, saves to database, triggers ingestion.

Logic:
1. Get the `code` from query params
2. Exchange it for an access token via `https://github.com/login/oauth/access_token`
3. Get the user's GitHub profile via `https://api.github.com/user`
4. Save to `github_profiles` table with status 'pending'
5. Trigger the ingestion job (call `/api/github/ingest` as a background job)
6. Redirect to `/dashboard/profile?github=connected`

**`app/api/github/ingest/route.ts`**
The heavy lifting. Fetches all repos, analyzes them with Claude, builds the technical fingerprint.

Logic:
1. Verify request has valid auth (either user session or CRON_SECRET header)
2. Fetch the candidate's github_profiles record
3. Update status to 'ingesting'
4. Call GitHub API: `GET /user/repos?sort=updated&per_page=100`
5. For each repo (limit to top 30 by recency and stars):
   - Fetch repo details
   - Fetch languages breakdown: `GET /repos/{owner}/{repo}/languages`
   - Fetch recent commits: `GET /repos/{owner}/{repo}/commits?per_page=10`
   - Fetch README: `GET /repos/{owner}/{repo}/readme` (base64 decode the content)
   - Fetch repo topics
6. Also fetch:
   - User's contribution stats: `GET /users/{username}/events/public?per_page=100`
   - Organizations: `GET /user/orgs`
7. Send all this data to Claude with the analysis prompt (see Prompts section below)
8. Parse Claude's JSON response
9. Update github_profiles with:
   - technical_fingerprint = Claude's output
   - repos_analyzed = processed repo list
   - ingestion_status = 'complete'
   - ingestion_completed_at = now()
10. Trigger autonomous matching for this candidate
11. Create a notification: "Your GitHub profile has been analyzed. Your agent is now actively matching you."

**`app/api/github/sync/route.ts`**
Re-runs ingestion for a candidate (called manually or on a weekly schedule). Same as ingest but updates existing record.

### 2. Job Market Crawler

**`app/api/crawler/run/route.ts`**
Fetches job postings from external sources and saves them to job_postings table.

This is the most complex route. Build it in phases:

**Phase 1 -- Greenhouse API (easiest, structured data)**
Greenhouse has a public API for job boards. Many companies use it.

```typescript
async function crawlGreenhouse() {
  // Greenhouse board tokens for major tech companies
  // These are public -- no auth required
  const boards = [
    { company: 'Stripe', token: 'stripe' },
    { company: 'Airbnb', token: 'airbnb' },
    { company: 'Figma', token: 'figma' },
    { company: 'Notion', token: 'notion' },
    { company: 'Vercel', token: 'vercel' },
    { company: 'Linear', token: 'linear' },
    { company: 'Retool', token: 'retool' },
    { company: 'Rippling', token: 'rippling' },
    // Add more as needed
  ]
  
  for (const board of boards) {
    const response = await fetch(
      `https://boards-api.greenhouse.io/v1/boards/${board.token}/jobs?content=true`
    )
    const data = await response.json()
    
    for (const job of data.jobs) {
      // Only process engineering roles
      if (!isEngineeringRole(job.title)) continue
      
      // Parse with Claude
      const parsed = await parseJobDescription(job.content, job.title, board.company)
      
      // Upsert to job_postings
      await supabase.from('job_postings').upsert({
        source: 'greenhouse',
        source_url: job.absolute_url,
        source_job_id: job.id.toString(),
        title: job.title,
        company_name: board.company,
        location: job.location?.name,
        raw_description: job.content,
        parsed_requirements: parsed,
        posted_at: job.updated_at,
        is_active: true
      }, { onConflict: 'source_url' })
    }
  }
}

function isEngineeringRole(title: string): boolean {
  const engineeringKeywords = [
    'engineer', 'developer', 'architect', 'devops', 'sre',
    'backend', 'frontend', 'fullstack', 'full-stack', 'full stack',
    'platform', 'infrastructure', 'data engineer', 'ml engineer',
    'software', 'technical lead', 'tech lead', 'staff engineer',
    'principal engineer'
  ]
  return engineeringKeywords.some(kw => title.toLowerCase().includes(kw))
}
```

### 3. Autonomous Matching Engine

**`app/api/match/run/route.ts`**
The core agentic loop. Runs continuously matching candidates against job postings.

Logic:
1. Verify CRON_SECRET in Authorization header
2. Get all candidates with:
   - github ingestion_status = 'complete'
   - agent_settings.auto_match_enabled = true
3. Get all active job postings from last 7 days
4. For each candidate + posting combination not already in autonomous_matches:
   - Calculate a quick pre-filter score (language overlap, location, comp range)
   - If pre-filter score > 30, run full Claude assessment
   - Save result to autonomous_matches
   - If score > agent_settings.notification_threshold, create notification
5. Return summary of matches created

**Pre-filter function** (run before expensive Claude call):
```typescript
function preFilterScore(candidate: CandidateProfile, githubFingerprint: any, posting: JobPosting): number {
  let score = 0
  
  // Language overlap
  const candidateLangs = githubFingerprint.primary_languages?.map((l: any) => l.language.toLowerCase()) || []
  const postingTechStack = posting.parsed_requirements?.tech_stack?.map((t: string) => t.toLowerCase()) || []
  const overlap = candidateLangs.filter((l: string) => postingTechStack.includes(l))
  score += overlap.length * 10
  
  // Seniority match
  const candidateYears = candidate.years_of_experience || 0
  const postingMin = posting.parsed_requirements?.years_experience_min || 0
  const postingMax = posting.parsed_requirements?.years_experience_max || 20
  if (candidateYears >= postingMin && candidateYears <= postingMax + 2) score += 20
  
  // Comp alignment
  if (candidate.comp_min && posting.parsed_requirements?.comp_max) {
    if (candidate.comp_min <= posting.parsed_requirements.comp_max) score += 15
  }
  
  // Remote preference
  if (candidate.remote_preference === 'remote_only' && posting.parsed_requirements?.remote_type === 'remote') score += 10
  if (candidate.remote_preference !== 'remote_only') score += 5
  
  // Visa
  if (candidate.visa_sponsorship_required && !posting.parsed_requirements?.visa_sponsorship) score -= 50
  
  return Math.max(0, Math.min(100, score))
}
```

### 4. Autonomous Outreach

**`app/api/outreach/draft/route.ts`**
Drafts an introduction email from the candidate's agent to the company.

Logic:
1. Takes an autonomous_match_id
2. Fetches the match, candidate profile, github fingerprint, and job posting
3. Calls Claude to draft a personalized introduction email
4. Saves draft to autonomous_matches.outreach_email_draft
5. Does NOT send -- candidate must approve first unless auto_outreach_enabled

The outreach email Claude prompt:
```
You are representing a software engineer as their career agent. Draft a concise, professional introduction email to the hiring team at {company} for the {role} position.

The email must:
- Be under 200 words
- Lead with the most compelling evidence from their GitHub work relevant to this role
- Reference 1-2 specific projects that demonstrate fit
- Not use generic phrases like "I am excited to apply"
- Sound like a human wrote it, not a bot
- Include a specific question about the role to show genuine interest
- End with a clear ask for a conversation

Candidate background: {summary}
Most relevant GitHub work: {github_evidence}
Role requirements: {key_requirements}
Fit assessment summary: {recommendation_summary}

Return ONLY the email body text, no subject line, no JSON wrapper.
```

**`app/api/outreach/send/route.ts`**
Actually sends the email. Requires explicit candidate approval.

For MVP use a simple email service -- Resend (https://resend.com) is the easiest to set up.
Add `RESEND_API_KEY` to environment variables.

export function buildAutonomousMatchPrompt(candidate: any, githubFingerprint: any, jobPosting: any): string {

## Scheduled Jobs (Cron)

Set up these endpoints to run on a schedule. For MVP use Vercel Cron Jobs (free on Vercel).

Add to `vercel.json` in the project root:

```json
{
  "crons": [
    {
      "path": "/api/crawler/run",
      "schedule": "0 */6 * * *"
    },
    {
      "path": "/api/match/run",
      "schedule": "0 */2 * * *"
    },
    {
      "path": "/api/github/sync",
      "schedule": "0 0 * * 0"
    }
  ]
}
```

This runs:
- Job crawler every 6 hours
- Matching engine every 2 hours
- GitHub re-sync weekly

Each endpoint must verify the `CRON_SECRET` in the Authorization header to prevent abuse:

```typescript
const authHeader = request.headers.get('authorization')
if (authHeader !== `Bearer ${process.env.CRON_SECRET}`) {
  return NextResponse.json({ error: 'Unauthorized' }, { status: 401 })
}
```
